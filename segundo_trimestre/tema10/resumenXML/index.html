Resumen XML


Es un lenguaje de marcas cuyo propósito es compartir datos a través de diferentes sistemas.

Las características de XML son:

  • Las etiquetas se definen
  • Consta de dos partes: Prólogo y Cuerpo
  • Utiliza espacios de nombres para evitar ambiguedades
  • Misma sintaxis que en HTML
  • Para que este bien formado, debe cumplir las reglas w3c
  • Usa esquemas en forma de arbol con una unica raiz
  • Se validan los documentos

Estructura documento XML

Consta de dos partes:

  • Prologo:
    1. Declara el XML
    2. Información del documento compuesta por:
      • Espacio de nombres
      • Hojas estilo XSLT
      • Hojas estilo CSS
  • Cuerpo:
    1. Arbol del contenido
    2. Unica raiz

Sintaxis XML

  • Elemento:
    • Es fundamental
    • Tiene que tener su etiqueta de apertura y de cierre
    • Contiene un atributo con su valor
    • Contiene un contenido
  • Elementos vacios:
    • Se define: < elemento />

Formacion de documentos

Normas básicas son:

  • Prólogo definido
  • Estructura de arbol con una unica raiz
  • Etiquetas emparejadas: apertura y cierre
  • Los atributos entre comillas
  • Se distingue las mayusculas de las minusculas
  • Las estiquetas se nombran
  • Los comentarios van fuera de las etiquetas

Validacion de documentos

Hay dos formas:

  1. DTD
  2. XML Schema

DTD

Describe la estructura,es decir, los elementos y el orden. Se puede ubicar de tres maneras distintas:

  1. Interna
  2. Externa
  3. Mixta

Elementos

Un ejemplo sería: < !ELEMENT elemento (hijo1, hijo2 ..) > < !ELEMENT elemento (#PCDATA) > Los elementos tienen tres características:

  • Cardinalidad: número de veces que aparece. Existen tres tipos:
    • ?: Aparece 0 o 1 vez
    • *: Aparece 0 a N veces
    • +: Aparece 1 a N veces
  • Secuencia: Indica los elementos que deben aparecer. Existen dos tipos:
    • X,Y: El elemento X tiene que aparecer a continuación del Y
    • X|Y: Aparece uno de los dos.
  • Combinación: Se pueden combinar la cardinalidad con la secuencia.

Atributos

Un ejemplo sería: < !ATTLIST elemento atributo tipo valor >, < !ATTLIST capitulo paginas CDATA #REQUIRED > Los atributos tienen varios tipos:

  • CDATA: Cadena de texto
  • Enumeraciones: Permite indicar varios valores
  • ENTITY: Entidad definida previamente.
  • ENTITIES: Conjunto de entidades.
  • IDREF: Se referencia el identificador de otro elemento.
  • IDREFS: Referencia un conjunto de identificadores.
  • ID: Identificador único para identificar elementos.
  • NOTATION: Se indican datos que no es XML.
  • NMTOKEN: Un nombre sin espacios en blanco.
  • NMTOKENS: Lista de nombres.
Los atributos también tienes varios valores:
  • Valor: El valor por defecto del atributo entre comillas.
  • #REQUIRED: El atributo es obligatorio.
  • #IMPLIED: El atributo es opcional.
  • #FIXED valor: El atributo es obligatorio y se le asigna un valor por defecto siendo el único que puede tener.

Entidades

Un ejemplo sería: < !ENTITY nombre_entidad valor >, < !ENTITY asignaturas "6" > Existen cinco entidades predefinidas:

  • <: Es el signo <.
  • >: Es el signo >.
  • ": Son las comillas rectas dobles, “.
  • ': Es el apóstrofe o comilla simple, ‘.
  • &: Es el ampersand, &.
