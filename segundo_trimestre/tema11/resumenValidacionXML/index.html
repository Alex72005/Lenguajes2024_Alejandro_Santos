Validación XML Schema

XML Schema (XSD) es una herramienta poderosa para definir reglas de validación en documentos XML. Mejora las capacidades de DTD al usar la sintaxis XML y permitir la personalización avanzada.

¿Qué es XML Schema?

Un esquema XML es un archivo que define:

  • Elementos y atributos permitidos.
  • Número y orden de elementos secundarios.
  • Tipos de datos para elementos y atributos.
  • Valores predeterminados y fijos.
Los esquemas XML tienen la extensión .xsd y permiten validar documentos XML instanciados.

Estructura de un XML Schema

Un esquema XML incluye:

  • Declaración de raíz mediante <xs:schema>.
  • Definición de elementos simples y complejos con <xs:element>.
  • Restricciones y tipos personalizados para datos específicos.

Elementos Simples y Complejos

Los elementos en XSD pueden ser:

  • Simples: Definidos con tipos predefinidos (e.g., xs:string, xs:integer) o personalizados mediante restricciones.
  • Complejos: Agrupaciones de elementos simples o estructuras anidadas, definidas con <xs:complexType>.

Atributos

Los atributos permiten añadir información adicional a los elementos:

  • Definidos con <xs:attribute>.
  • Permiten valores por defecto, opcionales o fijos.
  • Admiten restricciones como enumeraciones o límites de longitud.

Validación y Asociación

Los documentos XML se asocian al esquema mediante los atributos:

  • xmlns:xsi: Espacio de nombres XML Schema.
  • xsi:noNamespaceSchemaLocation: Ruta al archivo XSD.
Esto asegura que el documento cumpla con las reglas definidas en el esquema.
